## Outcome — short version

PR D (tracker DNT/GPC client-side check removal + server defaults flip) shipped to production cleanly after **three follow-up fixes**:
- PR #79: healthz probe budget 30s → 90s for Netcup cold-start path (LEARN.md Lesson 25)
- PR #80: deploy script must honor systemd env, not just YAML (LEARN.md Lesson 26)
- Manual symlink swing on Netcup since rc3's deploy fired before PR #80 was merged

**End-to-end smoke test PASSED**: a synthetic POST with `DNT: 1` + `Sec-GPC: 1` headers returns HTTP 202 and increments `received_total`. Pre-fix this would have been silently no-op'd by the tracker.

**The wp-slimstat.com 88% under-count was NOT primarily caused by the DNT/GPC client-side check.** Actual traffic is genuinely low (~5 events/day over 7 days). The "100 visits over 3 days" customer claim does not match either pre-fix or post-fix `/metrics` data — the customer was likely citing WP-Slimstat's own visit counts, which include bot crawls, prefetch hits, and other non-statnive-counted events.

**The real bug surfaced by `/metrics` instrumentation**: 77% of received events (55 of 71) are dropped as `hostname_unknown` — the user's NEW sites (`zh.statnive.com`, `ja.statnive.com`, `www.` variants of statnive.de / fr.statnive.com) aren't seeded in `statnive.sites`. **Operator action required** (admin API call + binary restart).

## Root-cause assessment

| Hypothesis | Evidence | Status |
|---|---|---|
| Tracker DNT/GPC client-side short-circuit | `received_total = 0` at T0 (clean baseline); after PR D the tracker now POSTs even for DNT/GPC visitors. Synthetic POST with both headers returns 202. | **Confirmed and fixed** by PR D |
| Server-side `dropped{hostname_unknown}` for new operator sites | 55/71 = 77% of post-fix events are dropped because the user's new mirrors aren't in `statnive.sites` | **NEW finding** — needs operator action |
| Bot detection eating real users | Last 5 wp-slimstat events all `is_bot=0` | **Not contributing** materially |
| wp-slimstat.com's claimed 100 visits over 3 days | Both pre-fix (12 events / 3 days) and post-fix (4 events / 41h) rates are ~3-5/day. WP-Slimstat plugin counts include sources statnive doesn't (bots, prefetch, partial page loads). | **Customer count is overstated** — apples-to-oranges comparison |
